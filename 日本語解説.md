# arcana v0.3.8

## 概要

- pythonラッパーの独自言語体系
- 以下の特徴を意識して開発
  - ラテン語ベースによる格式と神聖性・神秘性
  - ;による終了句の明示、{}によるスコープ可視化
  - 型安全の配慮
  - 哲学的思想体系の組込
  - python組込関数の再利用(ラップして独自体系化)

## 基本方針

**1. 宣言の明示・定義書の事前準備**

  - 引用/準備/経典の3セクションで記述し、省略してはならない
  - 経典には主題のみが存在し、別の主張があらわれてはならない
  - 事前準備を怠ってはならない
  - 先駆者の引用は敬意をもって示すこと
  - 各種宣言は必ず「何を」「どの形で」「どのように」を明示すること
  - 宣言に背くことを禁ずる

**2. 永劫回帰**

  - 人生は無限回の反復と考え、来る終わりを迎えることにあまえないこと
  - 輪廻のサイクルからは、意志による離脱のみ許される
  - 三度の転生のみ許され、それ以上の深淵に踏み込むことは許されない
  - 輪廻の輪は有限、ただし思索にふけるのは拒否されない

**3. 命題主義**

  - 常に命題を意識し、真であるか否かを判断すること
  - 真の否定は偽ではなく、￢真であり第3の判断はそのうえで行うこと
  - 真であるか否かを判別できるような命題を立てるよう心がけること

**4．力の指向性**
  
  - 複写と移動を明確に分けて考えること
  - 受け取る側と渡す側が明確にわかるようにすること
  - 流れの反転は、単に力の移動方向ではなく「意味の移動」ととらえること
  
## 詳細解説

### 全体

**Basic struct**

```arcana
/// インラインコメント
<FONS></FONS> /// 空でも必ず記載する
<INTRODUCTIO></INTRODUCTIO>　/// 空でも必ず記載する
<DOCTRINA>
FCON subjecto: nihil () -> { /// 必ず記載し省かないこと
    /// ... ここに処理を記載
}
</DOCTRINA>
```

**変数宣言**

 ```arcana
VCON Identifier: type = init_value;
Identifier = new_value; /// 〇 値の代入
Identifier <- new_value;  /// ×　変数のみ指定可能(式NG)
new_Identifier <- Identifier; /// 〇 このあとIdentifierは無効化
indicant () <- (Identifier); /// 引数の時は必ず()でくくる
VCON ordi_Identifier: ordinata = ordinata () <- (1, 2, 3); /// ordinata型
/// ordinata型の直接代入は非推奨、一度変数へ格納する
indicant () <- (ordi_Identifier[indexer]); /// indexerでどの値か指定
VCON catalogus_Identifier: catalogus = {"key": value}; /// catalogus型、初期宣言でのみ作成可
```

```bash
VCON: 変数宣言
Identifier: 変数名
type: 変数の型
init_value: 初期値(未定義でも可)

define_types: equivalent python types
inte: int
real: float
filum: str
verum: bool
ordinata: tuple
catalogus: dict
 ```

### 関数宣言

```arcana
FCON Identifier: type (*args:type) -> { statements }; /// 定義式：左辺を右辺と定義する
Identifier () <- (*args); /// 利用：左辺に右辺を渡す(空でも可、ただし()は必須)
```

```bash
FCON: 関数宣言
Identifier: 変数名
*args: 引数(複数指定可)
type: 変数の型
statement: 関数内で行う処理
```

### SI(if statement)

```arcana
SI propositio: ( expr ){
    VERUM{ statement }  /// True case：必ず記述
    FALSUM{ statement } /// false case：必ず記述
};
```

```bash
SI: if文宣言
propositio: 命題指定(expr=条件式)・必ずverum(boolean)であること
expr: 式
VERUM: Trueブロック、記載必須
FALSUM: Falseブロック、記載必須
```

### RECURSIO

```arcana
/// 内部的にwhile propositio==true、quotaによる上限100ループ
/// ユーザ指定によりループ数変更可
RECURSIO (propositio: ( expr ), quota: expr, acceleratio: expr) -> {
    statement
};

RECURSIO (propositio: ( expr ), quota: expr, acceleratio: expr) -> {
    statement
    RECURSIO (propositio: ( expr ), quota: expr, acceleratio: expr) -> {
        statement
        RECURSIO (propositio: ( expr ), quota: expr, acceleratio: expr) -> {
            statement /// レベル3ネストまではOK
            /// 制限：レベル4ネストはNG
            RECURSIO (propositio: ( expr ), quota: expr, acceleratio: expr) -> {
                statement
            };
        };
    };
};

RECURSIO (propositio: ( expr ), quota: expr, acceleratio: expr) -> {
    SI propositio: (...){
        VERUM{ effigium; }
        FALSUM{ proximum; }
    };
};

```

```bash
RECURSIO: ループ宣言
propositio: 命題指定(expr=条件式)・必ずverum(boolean)であること
quota: ループ上限、省略可、その場合はdefault:100
acceleratio: ステップバリュー、省略可、その場合はdefault: inner_counter += 1
effigium: break句
proximum: continue句
```
